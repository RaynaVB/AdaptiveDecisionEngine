# Import Pipeline

## Goal
Create a repeatable pipeline that can import:
- canonical ingredient seed rows
- alias rows
- dish ingredient priors
- future USDA / recipe / Open Food Facts staging outputs

## Recommended architecture

### Input sources
- manual CSV export from curation
- JSON files generated by upstream ETL
- HTTP-triggered admin import endpoint
- Cloud Storage file drop for larger batches

### Pipeline stages
1. **Create import job**
2. **Load input file or payload**
3. **Normalize row shape**
4. **Validate required fields**
5. **Resolve deterministic document IDs**
6. **Batch upsert canonical docs**
7. **Send unmapped phrases to unresolved queue**
8. **Write job stats and final status**

## Import job types
- `ingredient_seed_import`
- `alias_seed_import`
- `dish_prior_import`
- `usda_candidate_import`
- `recipe_phrase_import`
- `off_alias_import`

## Validation rules

### Ingredient row
Must contain:
- `ingredient_id`
- `canonical_name`
- `display_name`
- `category`

### Alias row
Must contain:
- `alias`
- `ingredient_id`

### Dish prior row
Must contain:
- `dish_id`
- `dish_name`
- `ingredient_id`

## Failure handling
- Row-level validation failures should not crash the whole job.
- Store errors in a job-scoped subcollection or array if the payload is small.
- Mark job `completed_with_errors` when needed.

## Idempotency
All imports should be idempotent.

Use:
- deterministic document IDs
- upsert semantics
- job metadata to avoid duplicate reprocessing of the same file if desired

## Large batch guidance
- Keep Firestore batches under 500 writes.
- Use chunk sizes of 250–400 for safety.
- For very large source files, parse into intermediate JSON and import in pages.

## Review loop for unresolved phrases
1. Import recipe or OFF phrase.
2. Try exact alias match.
3. Try normalized alias match.
4. Try canonical-name exact match.
5. If still unresolved, write to `unresolved_ingredient_phrases`.
6. Human reviewer maps phrase to canonical ingredient or marks as ignore.
7. Approved resolution creates a new `ingredient_aliases` doc.
