# Recommendation Feedback Schema

This document details the schema for persisting recommendation feedback in the Adaptive Decision Engine, completing part of Phase 3B. We use `AsyncStorage` to store this data locally on the device.

## Entity: `FeedbackEvent`

Feedback represents a user's action taken on a specific recommendation generated by the engine.

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier for this feedback event (UUID). |
| `recommendationId` | `string` | ID of the `Recommendation` this feedback applies to. |
| `recommendationType` | `string` | The category of recommendation (e.g., `substitution`, `timing_intervention`). |
| `outcome` | `FeedbackOutcome` | The explicitly chosen user outcome. |
| `timestamp` | `string` | ISO string denoting when the user submitted the feedback. |

### Valid `FeedbackOutcome` Values:
- `accepted_fully` ("Adopted ✅")
- `accepted_partially` ("Partially Accepted ⚠️")
- `rejected` ("Rejected ❌")

## Usage
Feedback data is consumed by:
1. **The UI Layer**: To show previous feedback states on recommendation cards.
2. **The Learning Logic**: The Recommender Engine computes cumulative rejection rates (`getRejectionRateByType`) to penalize poor-performing recommendation types in the future.
